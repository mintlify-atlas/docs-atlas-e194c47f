Linked modals synchronize with the browser's URL hash, enabling back/forward button navigation and deep linking to modal states.

## Complete Example

<CodeGroup>

```html HTML
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">

<body>

  <button data-srfc-open="about">Open About</button>
  <button data-srfc-open="contact">Open Contact</button>

  <div data-srfc="linked">

</html>
```

## How It Works

### The `linked` Type

Add `data-srfc="linked"` to the container to enable URL hash synchronization:

```html
<div data-srfc="linked">
  <div id="modal-name">...</div>
</div>
```

### URL Behavior

When a linked modal opens:
- URL changes to `#modal-id` (e.g., `https://example.com#about`)
- New history entry is pushed to browser history
- Page is shareable at this URL

When the modal closes:
- Browser navigates back, removing the hash
- `popstate` event is handled automatically

### Navigation Between Modals

Opening a linked modal while another linked modal is active:
- URL hash is **replaced** (not pushed) with the new modal ID
- Creates seamless navigation without cluttering history

### Back Button Integration

Pressing the browser's back button:
- Closes the current modal
- Returns to the previous state
- No page reload required

## Deep Linking

Visiting a URL with a hash automatically opens that modal:

```
https://example.com#contact → Opens the Contact modal
https://example.com#settings → Opens the Settings modal
```

<Tip>
Linked modals are perfect for forms, wizards, or any content you want users to bookmark or share.
</Tip>

## Progressive Enhancement

Use anchor links as openers for JavaScript-free fallback:

```html
<a href="#about" data-srfc-open="about">Open About</a>
```

The `data-srfc-open` attribute prevents default navigation and opens the modal instead.
