## How It Works

### The `arrow` Type

Add `data-srfc="arrow"` to enable keyboard arrow navigation:

```html
<div data-srfc="arrow">
  <div id="image-1">...</div>
  <div id="image-2">...</div>
  <div id="image-3">...</div>
</div>
```

### Keyboard Navigation

When a modal is open in an `arrow` group:
- **Arrow Right** / **Right**: Opens the next modal
- **Arrow Left** / **Left**: Opens the previous modal
- **Escape**: Closes the modal

### Sibling Detection

Srfc automatically detects adjacent modals within the same container:
- Modals navigate to their immediate sibling elements
- First modal has no previous
- Last modal has no next
- Only elements with an `id` attribute are considered modals

## Opening Specific Images

You can open any image programmatically:

```javascript
const srfc = new Srfc();
srfc.init();

// Open the third image
srfc.open('image-3');
```

## Combining with Linked

You can combine `arrow` with `linked` for URL integration:

```html
<div data-srfc="arrow linked">
  <div id="image-1">...</div>
  <div id="image-2">...</div>
</div>
```

Now the URL will update as users navigate: `#image-1`, `#image-2`, etc.

<Tip>
Arrow navigation works automatically based on DOM order. Arrange your modal elements in the order you want users to navigate through them.
</Tip>

<Note>
The arrow key listeners are only active when a modal in the `arrow` group is open, so they won't interfere with page scrolling or other keyboard interactions.
</Note>
